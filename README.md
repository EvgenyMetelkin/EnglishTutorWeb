# Репетитор английского

Одностраничное веб-приложение (SPA) для практики английского языка в формате чата.
В роли репетитора выступает локальная языковая модель (по умолчанию **Llama 3.2 3B**)
через [Ollama](https://ollama.com). Ответы приходят потоково и остаются короткими,
чтобы приложение работало быстро даже на слабом железе.

## Возможности

- Чат с репетитором и потоковой (streaming) выдачей ответов.
- Выбор любой локально установленной модели Ollama + кнопка обновления списка.
- 5 стилей репетитора (см. ниже), каждый со своим системным промтом.
- Выбор языка общения репетитора (русский / английский); при разборе ошибок и
  примерах английский текст сохраняется без перевода.
- Светлая / тёмная тема: следует системной, есть ручной переключатель.
- Сохранение настроек и истории чата в `localStorage` (переживают перезагрузку).
- Прерывание генерации: кнопка «Стоп», а также новый вопрос обрывает текущий ответ.
- Понятная обработка ошибок, если Ollama недоступен (сообщение в чате + тост).
- Ноль зависимостей, без сборки — чистые HTML/CSS/JS (ES-модули).

## Стили репетитора

| Стиль | Поведение |
|-------|-----------|
| Естественный диалог | Предлагает более живой вариант, если фраза звучит неестественно. Не трогает грамматику, если ошибок нет. |
| Исправление предложения | Только исправленный вариант, без комментариев. |
| Объяснение ошибки | Кратко объясняет, что было неверно (можно с правилом). |
| Краткий совет | Короткое правило или мнемоника, чтобы запомнить принцип. |
| Личный репетитор | Сам выбирает лучший подход и лаконично сочетает исправление / объяснение / совет / естественную альтернативу. |

## Требования

- [Ollama](https://ollama.com/download) (macOS, Linux или Windows).
- Хотя бы одна загруженная модель (рекомендуется `llama3.2:3b`).
- Любой современный браузер.
- Небольшой статический HTTP-сервер для раздачи файлов. Подойдёт встроенный
  сервер Python, `npx serve` или расширение Live Server в VS Code.

> **Важно:** приложение использует ES-модули (`import`) и `fetch`, поэтому его
> нужно открывать по `http://…`, а **не** двойным кликом (`file://`) — при
> `file://` браузер заблокирует загрузку модулей.

## Запуск локально

### 1. Установите и запустите Ollama

```bash
# Установка: https://ollama.com/download
ollama serve            # запустить сервер (обычно уже работает как сервис)
```

По умолчанию Ollama слушает `http://localhost:11434`.

### 2. Загрузите модель

```bash
ollama pull llama3.2:3b
```

Проверить список моделей:

```bash
ollama list
```

### 3. Разрешите запросы из браузера (CORS)

Браузер обращается к Ollama с другого origin (например, `http://localhost:8000`),
поэтому нужно разрешить CORS. Запустите Ollama с переменной окружения:

```bash
# macOS / Linux
OLLAMA_ORIGINS="*" ollama serve
```

```powershell
# Windows (PowerShell)
$env:OLLAMA_ORIGINS="*"; ollama serve
```

> На macOS, где Ollama работает как приложение, можно задать глобально:
> `launchctl setenv OLLAMA_ORIGINS "*"` и перезапустить Ollama.

### 4. Раздайте файлы приложения

Из папки проекта (там, где лежит `index.html`) запустите статический сервер:

```bash
# Вариант 1: Python 3
python3 -m http.server 8000

# Вариант 2: Node.js
npx serve -l 8000
```

### 5. Откройте приложение

Перейдите в браузере на:

```
http://localhost:8000
```

Выберите модель, стиль репетитора и язык общения в боковой панели и начинайте
писать фразы на английском.

## Использование

- **Отправка:** кнопка «Отправить» или `Enter` (перенос строки — `Shift+Enter`).
- **Стоп:** прерывает текущую генерацию, сохраняя уже полученный текст.
- **Обновить список моделей:** кнопка `⟳` рядом с выбором модели.
- **Язык общения:** «Русский» или «English»; влияет на язык пояснений, но не на
  разбираемый английский текст.
- **Очистить чат:** удаляет историю (настройки сохраняются).
- **Тема:** кнопка `◐` в шапке переключает светлую/тёмную тему.

## Структура проекта

```
.
├── index.html      # разметка: шапка, боковая панель настроек, чат, поле ввода
├── styles.css      # темы (CSS-переменные), адаптивная вёрстка, стили сообщений
├── prompts.js      # 5 системных промтов стилей (легко заменяемы)
├── app.js          # состояние, клиент Ollama, стриминг, рендер, обработка ошибок
└── docs/superpowers/   # проектные спецификация и план реализации
```

## Настройка

- **Модель по умолчанию:** `llama3.2:3b` — задаётся в `app.js` (`state.model`).
- **Язык по умолчанию:** русский (`DEFAULT_LANG` в `prompts.js`).
- **Адрес Ollama:** константа `OLLAMA_BASE` в `app.js` (`http://localhost:11434`).
- **Контекст диалога:** отправляются системный промт + последние 4 реплики (пары)
  + текущее сообщение (`TURN_WINDOW` в `app.js`).
- **Тексты промтов:** редактируются в `prompts.js` без изменения логики приложения.

## Возможные проблемы

| Симптом | Причина / решение |
|---------|-------------------|
| «Не удалось получить список моделей» | Ollama не запущен, либо не задан `OLLAMA_ORIGINS`. |
| Ошибки CORS в консоли браузера | Запустите Ollama с `OLLAMA_ORIGINS="*"`. |
| Пустая страница, ошибки про модули | Открыли через `file://`. Раздайте файлы по `http://`. |
| Модель не в списке | Загрузите её: `ollama pull <имя>`, затем нажмите `⟳`. |
| Ответы очень медленные | Модель тяжёлая для устройства — выберите модель поменьше (например, `3b`). |
